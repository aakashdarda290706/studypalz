Abstract

StudyPalz presents an innovative approach to personalized learning through an intelligent learning management system that implements advanced recommendation algorithms. The system employs a sophisticated hybrid approach, combining collaborative filtering with content-based recommendations to deliver highly tailored learning experiences. Through continuous analysis of student performance metrics, comprehensive tracking of knowledge gaps, and dynamic study schedule optimization, StudyPalz significantly enhances academic performance and learning outcomes. The system's architecture integrates cutting-edge user profiling techniques, adaptive assessment methodologies, and dynamic resource recommendation algorithms to create a responsive and effective personalized learning environment. Initial implementation results demonstrate significant improvements in student engagement and learning efficiency, with measurable gains in topic mastery and retention rates.

B. Learning Path Generation Algorithm

The learning path generation implements a modified A* algorithm with the following characteristics:

Algorithm 1: Adaptive Learning Path Generation
Input: User profile U, Knowledge state K
Output: Optimized learning path P

1: Initialize priority queue Q
2: Initialize visited set V
3: Initialize path cost C
4: For each topic t in curriculum:
5:    Calculate difficulty weight w(t)
6:    Calculate prerequisite relationships p(t)
7:    Add to topic graph G
8: While Q not empty:
9:    current = Q.pop()
10:   if current meets learning goal:
11:      return reconstruct_path(current)
12:   for neighbor in G[current]:
13:      if neighbor not in V:
14:         cost = C[current] + w(neighbor)
15:         heuristic = estimate_completion(neighbor)
16:         Q.push(neighbor, cost + heuristic)
17: return P

C. Assessment System

The assessment system employs Item Response Theory (IRT) combined with adaptive testing:

Fig. 4. Adaptive Assessment System Workflow

1) Question Selection Algorithm

Questions are selected using a modified version of the 3-Parameter Logistic Model (3PL):

P(θ) = c + (1-c) * [1 / (1 + e^(-1.7a(θ-b)))]

Where:

  • θ: Student ability level
  • a: Question discrimination
  • b: Question difficulty
  • c: Guessing parameter
